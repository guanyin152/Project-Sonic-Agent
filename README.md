# Project Sonic-Agent：面向沉浸式互动配音的音频工程自动化管线

## 项目定位与开发思路
本项目致力于解决互动音频社区（如戏精APP、剧本杀圈）中“官方做本难、后期门槛高”的痛点。我们的开发思路是“Code as Audio（音频即代码）”：弃用传统的手动剪辑软件，利用大模型将创意直接转化为结构化的时间轴工程文件。系统不生成台词人声，而是精准生成带有时间戳的专属BGM与环境音效，为玩家提供“拎包入住”式的沉浸式配音底噪环境。

## 核心架构与步骤化执行逻辑
1. **前置设定与世界观注入**：系统调用 RAG 模块，从本地向量数据库中抽取符合题材的音效库索引、BGM 风格指南。
2. **多智能体剧本与音效博弈**：编剧 Agent 与导演 Agent 介入审查，在台词间隙强制插入环境音提示，并进行多轮互相纠错。
3. **非结构化文本的 JSON 解析**：将剧本严格转换为包含 timestamp, audio_type 等标签的高阶 JSON 树。
4. **后端自动化执行**：Python 后端读取 JSON，利用 FFmpeg 拼接合成无台词纯沉浸伴奏轨。

# 核心演示案例：《狱》（高燃群像剧情歌自动对轨）

## 案例背景与指令
本案例展示系统如何通过一句指令，全自动生成带有精准时间轴的沉浸式“盲配”底噪工程。
**输入指令**："生成一段时长约2分钟的高燃奇幻剧情歌剧本及音效工程，主题为《狱》。包含背叛、绝地反击与生死羁绊。需要密集的法术、刀剑音效卡点，BGM需经历‘压抑-爆发-史诗-温情’四段式转折。输出 JSON 时间轴与玩家剧本。"

## 系统输出解析
* **BGM 情绪动态渲染**：00:30-00:35 绝对静音与心跳声；00:35 切入重金属打击乐。
* **核心音效精准卡点**：[00:58.000] 玩家念出“杀！”后，瞬间切入多重刀剑碰撞与法术爆炸声。
* **Token 消耗评估**：为了实现上述“台词情感”与“音效波形”的绝对契合，单次工程生成消耗超过 220K Tokens，具备极高工业并发测试价值。
